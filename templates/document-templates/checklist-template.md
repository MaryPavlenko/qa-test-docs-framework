# Checklist Template

## Checklist Information

| Field | Description |
|---|---|
| Feature | Name of the feature being tested |
| Module | Related application module |
| Environment | Web / Mobile / API |
| Tester | QA Engineer name |
| Date | Testing date |
| Version / Build | Application version |

---

# Positive Checks

- [ ] Verify expected behavior with valid input
- [ ] Verify successful user flow
- [ ] Verify correct UI behavior
- [ ] Verify navigation between related screens/pages

---

# Negative Checks

- [ ] Verify validation for empty required fields
- [ ] Verify validation for invalid input
- [ ] Verify error handling behavior
- [ ] Verify restricted or blocked actions

---

# Edge Cases

- [ ] Verify boundary values
- [ ] Verify behavior with long input values
- [ ] Verify behavior with special characters
- [ ] Verify repeated user actions
- [ ] Verify behavior after page refresh / app restart

---

# UI / UX Checks

- [ ] Verify layout alignment
- [ ] Verify text readability
- [ ] Verify button states
- [ ] Verify responsive behavior on different screen sizes
- [ ] Verify no overlapping UI elements

---

# Network / Performance Checks

- [ ] Verify behavior with slow internet connection
- [ ] Verify behavior without internet connection
- [ ] Verify loading indicators are displayed correctly
